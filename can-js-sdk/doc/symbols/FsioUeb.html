{object} createNotificationContext(device_id, auth_token, keep_alive_threshold, include_content_types, default_throttling_threshold, content_specific_thresholds)
Create notification context to be used with #waitForNotifications.
Parameters:
{string} device_id
Identifier of the device making the request.
{string} auth_token
Ticket must be provided only if the deployment in question has been configured to require authenticating notification requests.
{integer} keep_alive_threshold
Time (seconds) after which the server must respond. Constraints: 30..3600.
{string[]} include_content_types
Array of content types client wants to get notified of. E.g. ["fsio:file","fsio:metadata"].
{integer} default_throttling_threshold Optional
Default throttling threshold (seconds). Has no effect for the first request made but for subsequent requests a response to the client is not sent until the throttling threshold number of seconds have elapsed since the last response was sent.
{object[]} content_specific_thresholds Optional
Content specific throttling thresholds. Array of objects with ContentType and Threshold property. E.g. [{ContentType: "fsio:file", Threshold: 60}, {ContentType: "fsio:metadata", Threshold: 30}].
waitForNotifications(ctx, last_seen_notifications, complete, params, headers)
Retrieve notifications. If there are no notifications currently available, complete callback is called when one or more notifications are available or keep alive threshold expires.
